Why 81 events? (Double counting)

You're right to question this. Here's the breakdown for UK Week 1 2026 (Jan 5-11):

Event Count Should you count it?
configurator_submission 46 Yes
test_drive_media_page 25 Yes
form_submit 10 No - automatic duplicate
Total 81

Actual submission events: 46 + 25 = 71 (not 81)

The form_submit event is automatically generated by GA4 and duplicates the custom events above.

Why 71 events but only 41 leads?

Root cause: The tracking fires when users click the submit button, not when the form actually submits successfully.

Here's what happens when someone clicks "Send Request":

Step What happens
1 User clicks button → GA4 event fires (counted)
2 reCAPTCHA checks if user is human
3 Form validates all fields
4 Data sent to server → Lead created in ESS

If steps 2, 3, or 4 fail, the event is still counted in GA4 but no lead is created in ESS.

Example: We tested the form and clicked submit without completing reCAPTCHA. The GA4 event fired, but no lead was created. This explains the gap.
Fix: We will update the tracking to only fire after the form successfully submits (step 4), so GA4 events will match ESS leads. This will be done in the next GTM update.
